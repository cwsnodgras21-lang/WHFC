# Definition of Done

A task is complete when:

- The feature matches the request
- No unrelated files were changed
- TypeScript passes
- Build passes
- Database changes are documented
- Supabase RLS impact is considered
- API routes/server actions validate inputs
- Error states are handled
- Empty states are handled
- Loading states are handled where relevant
- Mobile layout is checked
- Tests or manual verification steps are provided
- Handoff summary is included
